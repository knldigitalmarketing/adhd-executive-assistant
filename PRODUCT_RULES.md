# Product Rules

## Core Rule

This is not a reminder app.

This is an executive-function assistant.

## Design Rules

- Do not rely on the user remembering everything.
- Ask how the user lives, then infer likely obligations.
- Use checklists instead of blank questions.
- Keep setup interviews short.
- Collect the big picture first.
- Fill in details progressively.
- Never block value because setup is incomplete.
- Every major life area should become a profile.

## Interview Rules

Initial interview should identify life structure:

- Home
- Work
- Transportation
- Health
- Family / Dependents
- Pets
- Finances
- Projects

Detailed interviews happen later only when relevant.

Example:

If user has a vehicle, later ask:

- Vehicle type
- Year
- Make
- Model
- Mileage
- Insurance
- Registration
- Maintenance needs

## Action System

The assistant should not only remind the user.

It should research and recommend the next action when possible.

Example:

Instead of:
"Your motorcycle tire needs replacement."

Say:
"Your motorcycle tire likely needs replacement. I found compatible options and the best installed price near you."

## Goal

Reduce memory load, decision fatigue, planning burden, and missed obligations.
